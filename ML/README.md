# 머신러닝 학습 로드맵 (상세)

처음부터 머신러닝을 배우는 사람을 위한 표준 학습 순서입니다. 각 챕터는 "무엇을 배우는가 → 핵심 개념 → 학습 포인트"로 구성했습니다. 체크박스로 진도를 관리하실 수 있습니다.

---

## 1. 수학·통계 토대

머신러닝의 모든 알고리즘은 결국 수학으로 표현됩니다. 다만 처음부터 수학을 완벽히 끝내고 시작할 필요는 없습니다. 큰 그림만 잡아두고, 각 알고리즘을 배울 때 필요한 부분으로 돌아오는 방식이 가장 효율적입니다.

**선형대수 (Linear Algebra)**

- [ ] 벡터와 행렬, 그리고 데이터가 왜 행렬로 표현되는가
- [ ] 행렬 곱, 전치(transpose), 역행렬
- [ ] 내적(dot product)과 그 기하학적 의미 (유사도·투영)
- [ ] 고윳값·고유벡터 (PCA, 차원 축소의 핵심)

**미적분 (Calculus)**

- [ ] 미분과 기울기(gradient)의 개념
- [ ] 편미분과 연쇄법칙 (신경망 역전파의 토대)
- [ ] 경사하강법(gradient descent)이 "왜" 작동하는가

**확률·통계 (Probability & Statistics)**

- [ ] 확률분포 (정규분포, 베르누이, 이항 등)
- [ ] 기댓값, 분산, 표준편차, 공분산, 상관계수
- [ ] 조건부확률과 베이즈 정리
- [ ] 가설검정, p-value, 신뢰구간
- [ ] 최대우도추정(MLE)의 직관

> **학습 포인트:** 증명을 따라가기보다 "이 개념이 어떤 알고리즘에서 쓰이는가"를 연결하며 익히세요. 예: 고윳값 → PCA, 베이즈 정리 → 나이브 베이즈, 경사 → 거의 모든 학습.

## 2. 프로그래밍과 도구

데이터를 직접 만지는 손기술을 익히는 단계입니다. 이론보다 키보드를 더 많이 두드려야 합니다.

- [ ] **Python 기초** — 자료구조, 함수, 클래스, 리스트 컴프리헨션
- [ ] **NumPy** — 배열 연산, 브로드캐스팅, 벡터화 (반복문 대신 행렬 연산으로 사고하기)
- [ ] **Pandas** — DataFrame 조작, 그룹화, 병합, 결측 처리
- [ ] **Matplotlib / Seaborn** — 분포·관계·추세 시각화
- [ ] **scikit-learn** — ML 표준 라이브러리, 일관된 `fit`/`predict` API
- [ ] **Jupyter Notebook** — 탐색적 분석과 실험 기록
- [ ] (선택) Git — 코드·실험 버전 관리

> **학습 포인트:** 데이터를 "벡터·행렬 연산"으로 다루는 사고방식이 핵심입니다. for문으로 한 줄씩 처리하던 습관을 NumPy/Pandas의 벡터화 연산으로 바꾸는 것이 첫 번째 도약입니다.

## 3. 머신러닝 기본 개념

알고리즘에 들어가기 전, 반드시 잡아야 할 큰 그림입니다. 이 챕터를 건너뛰면 나중에 모델이 왜 안 되는지 진단할 수 없게 됩니다.

**학습의 종류**

- [ ] 지도학습(supervised) — 정답이 있는 데이터로 학습 (회귀·분류)
- [ ] 비지도학습(unsupervised) — 정답 없이 구조를 찾음 (군집·차원축소)
- [ ] 강화학습(reinforcement) — 보상을 통해 행동을 학습 (개념만)

**문제 정의**

- [ ] 회귀(regression) vs 분류(classification)의 차이
- [ ] 특성(feature)과 타깃(target), 입력과 출력

**"학습"의 본질**

- [ ] 손실함수(loss function) — 모델이 얼마나 틀렸는지 측정
- [ ] 최적화(optimization) — 손실을 줄이는 방향으로 파라미터 조정
- [ ] 경사하강법의 직관적 이해

**모델의 일반화**

- [ ] 과적합(overfitting) vs 과소적합(underfitting)
- [ ] 편향-분산 트레이드오프(bias-variance tradeoff)
- [ ] 훈련/검증/테스트 분할 — 왜 데이터를 나누는가
- [ ] 데이터 누수(data leakage)의 위험성

> **학습 포인트:** "모델이 훈련 데이터를 잘 맞히는 것"과 "처음 보는 데이터를 잘 맞히는 것"은 전혀 다른 문제라는 점이 이 챕터의 핵심입니다. ML의 목표는 후자(일반화)입니다.

## 4. 데이터 전처리와 특성 공학

현업 ML 작업의 시간 대부분이 여기서 소요됩니다. "좋은 데이터가 좋은 모델을 만든다"는 말이 가장 잘 적용되는 단계입니다.

**데이터 정제**

- [ ] 결측치(missing value) 처리 — 삭제 / 대치(평균·중앙값·보간)
- [ ] 이상치(outlier) 탐지 — IQR, z-score, 시각적 탐지
- [ ] 중복 제거, 데이터 타입 정합성

**변환과 스케일링**

- [ ] 정규화(Min-Max)와 표준화(StandardScaler)
- [ ] 로그 변환 등 분포 보정
- [ ] 범주형 인코딩 — One-Hot, Label, Target Encoding

**특성 공학(Feature Engineering)**

- [ ] 도메인 지식 기반 특성 생성
- [ ] 특성 선택(feature selection)과 중요도
- [ ] 다중공선성(multicollinearity) 점검

**불균형 데이터**

- [ ] 클래스 불균형 문제 (예: 사기 탐지)
- [ ] 오버샘플링/언더샘플링, SMOTE
- [ ] 클래스 가중치(class weight) 조정

> **학습 포인트:** 전처리 규칙은 반드시 "훈련 데이터에서 학습 → 테스트 데이터에 적용" 순서를 지켜야 합니다. 테스트 데이터의 통계(평균 등)를 미리 보고 스케일링하면 데이터 누수가 발생합니다.

## 5. 핵심 알고리즘 — 지도학습

ML의 본체입니다. 각 알고리즘을 **개념 → 수식 → 코드 구현** 순서로 익히면 직관과 실전이 동시에 잡힙니다.

- [ ] **선형회귀** — 가장 기본, 최소제곱법의 이해
- [ ] **다항회귀·정규화** — Ridge(L2), Lasso(L1), 과적합 제어
- [ ] **로지스틱 회귀** — 분류의 출발점, 시그모이드와 확률
- [ ] **결정트리(Decision Tree)** — 분기 규칙, 지니/엔트로피, 해석력
- [ ] **랜덤포레스트** — 배깅(bagging) 기반 앙상블
- [ ] **그래디언트 부스팅** — XGBoost, LightGBM, CatBoost (표 형식 데이터의 실무 주력)
- [ ] **k-NN** — 가장 직관적인 거리 기반 분류
- [ ] **나이브 베이즈** — 베이즈 정리의 응용
- [ ] **SVM** — 마진 최대화, 커널 트릭 (개념 위주)

> **학습 포인트:** 모든 알고리즘을 깊게 팔 필요는 없습니다. 선형/로지스틱 회귀로 기초를 잡고, 트리 기반 앙상블(특히 부스팅)을 실무 주력으로 깊이 익히는 것이 현실적입니다.

## 6. 모델 평가와 검증

만든 모델이 정말 쓸 만한지 객관적으로 판단하는 도구들입니다. 잘못된 지표를 쓰면 나쁜 모델을 좋은 모델로 착각하게 됩니다.

**회귀 지표**

- [ ] MAE(평균절대오차), MSE/RMSE(제곱오차), R²(결정계수)

**분류 지표**

- [ ] 정확도(accuracy)와 그 한계 (불균형 데이터에서 함정)
- [ ] 정밀도(precision)·재현율(recall)·F1 score
- [ ] 혼동행렬(confusion matrix) 읽는 법
- [ ] ROC 곡선과 AUC, PR 곡선

**검증 전략**

- [ ] 홀드아웃(hold-out) 분할
- [ ] k-fold 교차검증(cross-validation)
- [ ] 계층적(stratified) 분할 — 클래스 비율 유지
- [ ] 시계열 데이터용 시간 기반 분할 (무작위 분할 금지)

**튜닝**

- [ ] 하이퍼파라미터 vs 파라미터의 차이
- [ ] Grid Search, Random Search, Bayesian Optimization

> **학습 포인트:** 문제에 맞는 지표를 고르는 것이 핵심입니다. 예를 들어 99%가 정상인 데이터에서는 "전부 정상"이라 찍어도 정확도 99%가 나오므로, 이런 경우 정밀도·재현율이나 PR-AUC를 봐야 합니다.

## 7. 비지도학습

정답(라벨)이 없는 데이터에서 숨은 구조를 찾는 방법입니다.

**군집화(Clustering)**

- [ ] k-means — 가장 기본, 중심 기반 군집
- [ ] 계층적 군집(hierarchical clustering)
- [ ] DBSCAN — 밀도 기반, 임의 모양 군집과 노이즈 처리

**차원 축소(Dimensionality Reduction)**

- [ ] PCA — 주성분 분석, 분산 보존
- [ ] t-SNE / UMAP — 고차원 데이터의 2D 시각화

**이상 탐지(Anomaly Detection)**

- [ ] Isolation Forest, One-Class SVM
- [ ] 정상 패턴에서 벗어난 데이터 식별

> **학습 포인트:** 비지도학습은 "정답이 없다"는 특성 때문에 결과 평가가 어렵습니다. 군집 개수 선택(엘보우 방법, 실루엣 점수)처럼 결과를 해석하는 기준을 함께 익혀야 합니다.

## 8. 딥러닝 기초

신경망으로 넘어가는 단계입니다. 데이터가 많고 패턴이 복잡할 때 빛을 봅니다.

**신경망의 기초**

- [ ] 퍼셉트론, 다층 신경망(MLP)의 구조
- [ ] 활성화 함수(ReLU, sigmoid, softmax)
- [ ] 순전파(forward)와 역전파(backpropagation)
- [ ] 경사하강법의 변형 (SGD, Adam 등)

**주요 아키텍처 (개념 위주)**

- [ ] CNN — 이미지·공간 데이터
- [ ] RNN / LSTM / GRU — 순차·시계열 데이터
- [ ] Transformer와 어텐션 — 현대 모델(LLM 포함)의 근간

**프레임워크**

- [ ] PyTorch (연구·실무 표준) 또는 TensorFlow/Keras
- [ ] GPU 활용, 배치 학습, 학습률 스케줄링

> **학습 포인트:** 딥러닝부터 시작하지 마세요. 데이터가 적거나 표 형식이면 트리 기반 모델이 더 강하고 빠릅니다. 신경망은 "왜 필요한가"를 이해한 뒤 도구로 추가하는 것이 좋습니다.

## 9. 실전과 MLOps

배운 것을 실제로 작동시키는 마무리 단계입니다. "노트북에서 되는 모델"과 "현업에서 굴러가는 모델"은 다릅니다.

- [ ] **재현성** — 랜덤 시드 고정, 실험 기록·추적
- [ ] **데이터 누수 방지** — 검증 설계의 함정 피하기
- [ ] **모델 해석** — SHAP, 특성 중요도, 부분의존도(PDP)
- [ ] **모델 배포** — API 서빙, 컨테이너화 기초
- [ ] **모니터링** — 데이터 드리프트, 성능 저하 감지
- [ ] **재학습 파이프라인** — 자동화된 학습·평가
- [ ] **종합 프로젝트** — 데이터 수집부터 배포까지 직접 완주

> **학습 포인트:** 가장 중요한 단계입니다. 작은 데이터셋이라도 "수집 → 전처리 → 모델링 → 평가 → 배포"의 전 과정을 직접 한 번 완주하면, 흩어져 있던 개념들이 하나로 묶입니다.

## 전체를 관통하는 학습 원칙

1. **이론과 실습을 분리하지 말 것.** 알고리즘 하나를 배우면 즉시 작은 데이터에 적용해 보는 사이클이 가장 빠르게 굳습니다.

2. **한 번에 다 배우려 하지 말 것.** 권장 동선은 다음과 같습니다.
   - **1차 순환 (전체 그림):** 3 → 5 → 6 으로 ML의 핵심 흐름을 빠르게 한 바퀴
   - **2차 순환 (기반 보강):** 1·2·4 로 토대와 전처리를 다지기
   - **3차 순환 (확장):** 7 → 8 → 9 로 비지도·딥러닝·실전으로 확장
   - 1(수학)은 막히는 부분이 생길 때마다 그때그때 돌아오기

3. **손실함수·일반화·검증, 이 세 개념이 전부의 뿌리.** 어떤 알고리즘을 배우든 이 세 가지로 환원해서 이해하면 길을 잃지 않습니다.
