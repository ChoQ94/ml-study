# 3강. "학습"의 본질 — 손실함수 · 최적화 · 경사하강법

이 강의가 챕터 3에서 가장 중요합니다. **"모델이 학습한다"** 는 추상적인 말의
정체를 수학적으로 벗겨냅니다. 이걸 이해하면 거의 모든 머신러닝 알고리즘이
같은 골격을 공유한다는 것을 알게 됩니다.

---

## 0. 학습 = "틀린 정도를 줄여나가는 과정"

모델은 내부에 **파라미터(parameter)** 라는 숫자 다이얼들을 가지고 있습니다.
학습이란 결국 이 다이얼을 **이리저리 돌려서 예측이 정답에 가까워지게 만드는 것**
입니다. 그 과정은 세 부품으로 이루어집니다.

```
        ┌─────────────────────────────────────────────┐
        │  ① 손실함수: 지금 얼마나 틀렸나? (점수 측정)   │
        │            ↓                                  │
        │  ② 최적화:  어느 방향으로 다이얼을 돌릴까?     │
        │            ↓                                  │
        │  ③ 경사하강: 그 방향으로 조금씩 반복해서 이동  │
        └─────────────────────────────────────────────┘
```

---

## 1. 손실함수 (Loss Function) — "얼마나 틀렸는가"

> **모델의 예측이 정답에서 얼마나 벗어났는지를 하나의 숫자로 나타낸 함수.**

손실이 **작을수록 좋은 모델**입니다. 학습의 목표는 이 손실을 최소화하는 것.

### 회귀의 대표 손실: 평균제곱오차 (MSE)

예측값 ŷ(와이햇)와 실제값 y의 차이를 제곱해서 평균낸 값입니다.

```
MSE = (1/n) · Σ (yᵢ − ŷᵢ)²
```

- `yᵢ − ŷᵢ` : 한 샘플의 오차 (정답 − 예측)
- **제곱하는 이유**: ① 부호를 없애 +오차와 −오차가 상쇄되지 않게 하고
  ② 큰 오차에 더 큰 벌점을 줘서 "크게 틀리는 것"을 강하게 막습니다.

**숫자로 느껴보기** — 정답이 `[10, 20, 30]`일 때

| 예측 | 오차 | 오차² | MSE |
|------|------|-------|-----|
| `[11, 19, 33]` | `1, -1, 3` | `1, 1, 9` | (1+1+9)/3 = **3.67** |
| `[10, 20, 30]` | `0, 0, 0` | `0, 0, 0` | **0** (완벽) |

### 분류의 대표 손실: 교차 엔트로피 (Cross-Entropy)

분류는 보통 "악성일 확률 0.87" 같은 확률을 출력합니다. 교차 엔트로피는
**정답 클래스에 높은 확률을 줬으면 손실↓, 낮은 확률을 줬으면 손실↑** 이 되도록
설계된 손실입니다. (자세한 식은 5강 로지스틱 회귀에서 다룹니다. 지금은
"분류에는 분류용 손실이 따로 있다"만 기억하세요.)

> 💡 **핵심:** 문제(회귀/분류)마다 어울리는 손실함수가 있습니다. 손실함수를
> 정하는 순간, "좋은 모델"의 정의가 수학적으로 확정됩니다.

## 2. 최적화 (Optimization) — "손실을 줄이는 방향 찾기"

손실함수가 "현재 점수"를 알려준다면, 최적화는 **"점수를 낮추려면 다이얼을
어느 쪽으로 돌려야 하나"** 를 푸는 문제입니다.

파라미터를 바꾸면 손실이 바뀝니다. 그래서 손실은 **파라미터의 함수**입니다.
이 함수를 그려보면 골짜기(valley) 모양이 되고, 우리는 **골짜기의 가장 낮은
지점**을 찾고 싶습니다.

```
손실 ↑
  │ \                         /
  │  \                       /
  │   \                     /
  │    \___              ___/
  │        \___      ___/
  │            \____/  ← 여기! (손실 최소 지점)
  └──────────────────────────→ 파라미터
```

문제는 우리가 이 그래프 전체를 한눈에 볼 수 없다는 것입니다.
파라미터가 수백만 개면 차원이 너무 높습니다. 그래서 **현재 위치에서 한 걸음씩
내려가는** 방법이 필요합니다. 그게 경사하강법입니다.

---

## 3. 경사하강법 (Gradient Descent) — "기울기를 따라 한 걸음씩"

> **안개 낀 산에서 눈을 가린 채 가장 낮은 골짜기로 내려가는 법:**
> **발밑의 기울기를 더듬어, 가장 가파르게 내려가는 쪽으로 한 걸음씩 내딛는다.**

여기서 "기울기"가 바로 **그래디언트(gradient)** = 손실함수의 미분값입니다.
(로드맵 1번 미적분이 여기서 연결됩니다.)

### 업데이트 규칙

```
새 파라미터 = 현재 파라미터 − (학습률 × 기울기)

      θ  ←  θ  −  η · ∇L(θ)
```

- `∇L(θ)` (그래디언트) : 손실이 가장 **가파르게 증가**하는 방향
- 그 **반대 방향**(`−`)으로 가야 손실이 줄어듭니다
- `η`(에타, **학습률 learning rate**) : 한 걸음의 크기

### 왜 이게 작동하는가

기울기가 **양수**면(오른쪽이 오르막) → 파라미터를 **왼쪽**(감소)으로 옮기고,
기울기가 **음수**면(오른쪽이 내리막) → 파라미터를 **오른쪽**(증가)으로 옮깁니다.
어느 쪽이든 항상 **손실이 줄어드는 방향**으로 이동합니다. 이걸 수천~수만 번
반복하면 골짜기 바닥(손실 최소)에 가까워집니다.

### 학습률(η)이라는 결정적 손잡이

| 학습률 | 증상 |
|--------|------|
| 너무 작음 | 한 걸음이 너무 작아 **수렴이 한없이 느림** |
| 적절함 | 안정적으로 골짜기로 내려감 ✅ |
| 너무 큼 | 골짜기를 **건너뛰며 튕겨서 발산**(손실이 오히려 폭발) |

> ⚠️ 학습률은 머신러닝에서 가장 자주 만지는 **하이퍼파라미터**입니다.
> "학습이 안 된다" 싶을 때 가장 먼저 의심할 후보 중 하나입니다.
> (하이퍼파라미터 개념은 로드맵 6번에서 다룹니다.)

### 변형들 (이름만 알아두기)

- **배치 경사하강법(BGD)** — 매 걸음마다 전체 데이터로 기울기 계산 (정확하지만 느림)
- **확률적 경사하강법(SGD)** — 샘플 하나(또는 소량)로 계산 (빠르고 흔들리지만 실전 주력)
- **Adam 등** — 학습률을 자동 조절해주는 똑똑한 변형 (딥러닝 기본값, 로드맵 8번)

## 4. 전체 그림으로 다시 보기

거의 모든 지도학습은 이 루프를 돕니다.

```
1. 파라미터를 아무 값으로 초기화한다
2. 반복:
     a. 현재 파라미터로 예측한다           → ŷ
     b. 손실함수로 얼마나 틀렸는지 잰다     → L
     c. 기울기(그래디언트)를 계산한다       → ∇L
     d. 기울기 반대 방향으로 파라미터를 옮긴다 → θ ← θ − η·∇L
3. 손실이 충분히 작아지면 멈춘다
```

> 💡 **이 루프가 챕터 3의 보물입니다.** 선형회귀든 신경망이든, 알고리즘이
> 달라지는 건 주로 "예측하는 방식(a)"과 "손실의 모양(b)"일 뿐, **"손실을 정의하고
> 경사로 내려간다"는 골격은 거의 그대로** 입니다. 이걸 잡으면 새 알고리즘을
> 만나도 "손실은 뭐고, 뭘 최소화하나?"라는 한 질문으로 꿰뚫어 볼 수 있습니다.
